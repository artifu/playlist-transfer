Spotify playlist link not loading?

Start with the URL and access test. Most preview failures happen before matching begins, so changing Apple Music settings will not fix them.

PlaylistXfer editorial · Last reviewed July 22, 2026

Fastest diagnosis

Open the playlist link in a signed-out private window. If Spotify does not show the playlist there, PlaylistXfer probably cannot read it through the public-link flow.

On this page Check the URL Check public access Understand the failure Retry safely Report a reproducible issue

1. Confirm that the link identifies a playlist or track

PlaylistXfer accepts normal Spotify playlist and track links. A playlist URL generally contains open.spotify.com/playlist/; a single song contains open.spotify.com/track/.

Link type Expected result
Playlist or track Supported by the current preview flow
Album, artist, profile, search, or queue Not a playlist transfer source
Personalized home or mix page May depend on your Spotify session and may not expose a stable public playlist
Old shared-private link May be restricted or expired

When in doubt, return to the playlist inside Spotify and use Share → Copy link to playlist. That avoids copying a surrounding browser page.

2. Test access outside your account

  1. Open a private or incognito window. Do not sign into Spotify.
  2. Paste the exact link. Wait for the playlist page to finish loading.
  3. Look for the title and track list. A login wall, access request, empty page, or redirect indicates the link is not publicly readable.
  4. Copy a fresh link if needed. Spotify says some shared links can expire, so a new link may behave differently.

If you own the playlist, review its visibility settings. If you do not own it, you may need a different public playlist rather than trying to bypass the owner’s access choice.

3. Separate link problems from service problems

A
“Enter a valid Spotify playlist or track URL”

The pasted value is empty, malformed, or points to another Spotify page type. Copy a fresh playlist or track link.

B
The link opens for you but not signed out

Your Spotify session has access that PlaylistXfer does not. Change visibility only if you own the playlist and are comfortable making it readable.

C
“Transfer API did not respond”

The PlaylistXfer service is temporarily unavailable or misconfigured. Keep the URL, wait briefly, and retry rather than changing the playlist.

D
Preview works but matching fails

The Spotify link was read successfully. The problem is now in the Apple Music matching stage, not the source URL.

4. Retry without creating duplicate work

  1. Keep one browser tab and paste a freshly copied link.
  2. Preview again before starting Apple Music matching.
  3. If the error names the Transfer API, wait a minute and retry once.
  4. If the same public link repeatedly fails, test a different known-public playlist. This separates a link-specific problem from a system-wide outage.
  5. Do not authorize Apple Music to solve a preview error. Authorization is only relevant at the final creation step.

5. Report an issue without sharing secrets

A useful report includes the step that failed, the exact visible error, browser and device, approximate time, and whether the link passed the signed-out test. A public playlist URL can help reproduce the issue, but do not post a private link if its content is sensitive.

Never include Apple Music tokens, cookies, account emails, passwords, or browser storage values. PlaylistXfer support does not need them to diagnose a public preview failure.

Before reporting:

Try one other public playlist. “This link fails but another works” is far more diagnostic than “nothing works.”
